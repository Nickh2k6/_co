第九章（Chapter 9）標誌著這門課程的一個重要轉折點。在前幾章中，您一直在構建硬體和底層軟體（組譯器、VM 翻譯器），而從第九章開始，您將進入**高階語言（High-Level Language）**的領域。

這一章的主題是 **「高階語言程式設計 (High-Level Language)」**。根據講義與專案說明，本章的核心內容如下：

### 1. 學習 Jack 語言
本章介紹了一種名為 **Jack** 的高階程式語言。
*   **特性：** Jack 是一種類似 Java 的、基於物件（Object-based）的語言,。它支援類別（Classes）、方法（Methods）、函數（Functions）、建構子（Constructors）以及陣列和字串等資料結構。
*   **語法：** 您將學習 Jack 的基本語法，例如變數宣告、`if/while` 流程控制、以及如何操作物件。

### 2. 專案目標：編寫一個應用程式 (Project 9)
本章的作業（Project 9）非常有創意：您需要用 Jack 語言發明並編寫一個電腦遊戲或互動程式。
*   **範例：** 講義提到的例子包括俄羅斯方塊（Tetris）、貪吃蛇（Snake）、乒乓球（Pong）、推箱子（Sokoban）等經典遊戲,。
*   **流程：**
    1.  編寫 `.jack` 原始碼檔案（通常包含 `Main.jack` 和其他類別檔案）。
    2.  使用課程提供的 **Jack 編譯器（Jack Compiler）** 將其編譯成 `.vm` 檔案。
    3.  在 **VM 模擬器（VM Emulator）** 中執行並測試您的遊戲,。

### 3. 使用 Jack OS（標準函式庫）
在編寫程式時，您不需要從零開始控制硬體。Jack 提供了一套類似作業系統的高階標準函式庫（Jack OS），您將學習如何使用這些 API：
*   **Screen：** 用於繪圖（畫點、線、圓、矩形）。
*   **Keyboard：** 處理鍵盤輸入。
*   **Output：** 處理文字輸出。
*   **Math / String / Array / Memory：** 處理數學運算、字串操作、陣列與記憶體管理,。

### 4. 記憶體管理概念
值得注意的是，Jack 語言（與 Java 不同）沒有自動垃圾回收（Garbage Collection）。因此，本章強調了手動管理記憶體的重要性。當您創建物件（使用 `new`）後，必須負責在不再使用時將其銷毀（使用 `dispose` 方法呼叫 `Memory.deAlloc`），以避免記憶體洩漏,。

### 5. 承先啟後的戰略意義
這一章看起來像是「休息」去寫遊戲，但其實有一個 **「隱藏議程 (Hidden Agenda)」**：
*   **為了寫編譯器（Project 10-11）：** 在您親手構建 Jack 的編譯器之前，您必須先成為 Jack 語言的熟練使用者，了解它的語法結構和行為邏輯。
*   **為了寫作業系統（Project 12）：** 您現在**使用**的 Jack OS 函式庫（如 `Screen.drawPixel`），在最後一章將由您親手用 Jack 語言**實作**出來,。
